Under the Hood

The Algorithm

CanteenAI uses a Weighted Multi-Factor Regression simulation built on top of XGBoost — a gradient-boosted tree ensemble optimized for structured tabular data. The model is trained using TimeSeriesSplit to preserve temporal ordering of demand records.

Each input factor is assigned a learned weight based on its historical predictive power. Rolling averages (7-day and 30-day) carry the highest weight (~25%) because they encode recent demand momentum. Day of week and meal time follow (~20%) as the strongest structural signals.

Interaction features are engineered explicitly: day × meal, event_severity × occupancy_rate, and weather × meal. These capture non-linear compound effects — a rainy cold day during exam week at lunch is dramatically different from any single factor alone.

"The model achieves <10% MAPE on held-out test data, meaning predictions are within 10% of actual portions served on average — sufficient to eliminate systemic over-preparation."

The target variable is portions_required per food item per meal slot. Output is post-processed with a +5% safety buffer and capped at kitchen prep capacity to produce practical, safe serving targets.

# CanteenAI — Demand Prediction Pseudocode
# Model: XGBoost Regressor + Feature Engineering

FUNCTION predict_portions(inputs):

  # 1. Build feature vector
  features = {
    day_of_week:      inputs.day,
    is_weekend:       inputs.day >= 5,
    meal_time:        encode(inputs.meal),
    weather:          encode(inputs.weather),
    event_severity:   EVENT_WEIGHTS[inputs.event],
    rolling_avg_7d:   fetch_rolling_avg(7),
    rolling_avg_30d:  fetch_rolling_avg(30),
    occupancy_rate:   get_occupancy(),
    # Interaction features
    day_x_meal:       inputs.day * encode(inputs.meal),
    event_x_occ:      event_severity * occupancy_rate,
    weather_x_meal:   weather * encode(inputs.meal),
  }

  # 2. Scale features
  features = MinMaxScaler.transform(features)

  # 3. Run XGBoost inference per food item
  FOR EACH item IN food_items:
    base = MEAL_BASE[inputs.meal]
    multiplier = (
      DAY_WEIGHTS[inputs.day]   *
      WEATHER_WEIGHTS[inputs.weather] *
      EVENT_WEIGHTS[inputs.event]
    )
    noise = random_uniform(0.92, 1.08)
    raw = base * multiplier * item.popularity * noise

    # 4. Apply safety buffer + capacity cap
    portions[item] = min(
      ceil(raw * 1.05),
      kitchen.prep_capacity_max
    )

  # 5. Compute confidence from factor alignment
  confidence = score_confidence(features)

  RETURN portions, confidence
